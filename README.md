## Operating Style

<div align="center">

| Principle | Meaning |
|---|---|
| **Business-first** | Features must solve real operational problems |
| **Premium but usable** | UI should look strong without damaging usability |
| **Secure by default** | Auth, roles, sessions, and access checks matter |
| **Scalable direction** | Demo first, then refactor toward multi-tenant SaaS |
| **Reliable workflows** | Billing, KOT, records, and reports must not be fragile |
| **Clean handoff** | Code and product flow should be understandable later |
